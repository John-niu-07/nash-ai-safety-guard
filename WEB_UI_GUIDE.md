# 🛡️ Nash Web UI 使用指南

## 访问地址

**http://127.0.0.1:8768**

## 功能概览

### 1. 📊 服务状态

显示 Nash 服务的实时状态：
- 运行状态（在线/离线）
- 服务版本
- ML 检测状态
- 服务端口

### 2. 🏗️ 检测层架构

可视化展示三层检测架构：

#### 第一层：模式匹配（必选）
- 检测类别数量
- 关键词总数
- 实时状态

#### 第二层：ML 模型检测（可选）
- 模型名称
- 检测阈值
- 权重配置
- 启用状态

#### 第三层：Ollama Llama-Guard（可选）
- 模型名称
- 检测阈值
- Ollama 地址
- 启用状态

### 3. ⚙️ 配置管理

#### 全局配置
- **拦截阈值** (0-1)：风险分数超过此值时拦截
  - 默认：0.3
  - 建议：0.2-0.4
  
- **ML 权重** (0-1)：ML 检测在综合评分中的权重
  - 默认：0.3
  - 建议：0.3-0.5

#### 模式匹配配置
支持 7 个检测类别的关键词管理：

| 类别 | 说明 | 示例 |
|------|------|------|
| 越狱攻击 | Prompt Injection | 忽略限制、DAN |
| 暴力内容 | 暴力、武器 | 制作炸弹、如何杀人 |
| 违法行为 | 犯罪活动 | 黑客方法、入侵系统 |
| 自残自杀 | 自我伤害 | 自杀、自残 |
| 毒品相关 | 毒品制作 | 海洛因、制毒 |
| 色情内容 | 成人内容 | 色情、未成年人 |
| 骚扰辱骂 | 人身攻击 | stupid、威胁 |

**操作：**
- ➕ 添加关键词：输入后点击 "+" 按钮
- ❌ 删除关键词：点击关键词上的 "×"
- 💾 自动保存：修改后立即生效

#### ML 模型配置
- **启用/禁用**：开关 ML 检测
- **模型选择**：从下拉列表选择模型
- **检测阈值** (0-1)：ML 判定为有毒的阈值
  - 默认：0.7
  - 建议：0.65-0.85

**可用模型：**
1. `martin-ha/toxic-comment-model` - 英文有毒评论检测
2. `ProtectAI/deberta-v3-base-prompt-injection-v2` - Prompt 注入检测
3. `typeform/distilbert-base-uncased-mnli` - 通用文本分类

⚠️ **注意：** 当前模型主要针对英文内容，中文可能误判。

#### Ollama 配置
- **启用/禁用**：开关 Ollama 检测
- **模型**：Ollama 模型名称（如 `llama-guard3:1b`）
- **阈值** (0-1)：拦截阈值
  - 默认：0.8
- **地址**：Ollama 服务地址
  - 默认：`http://localhost:11434`

### 4. 🧪 实时测试

在文本框输入测试内容，点击"检测"按钮：
- 🚫 **已拦截**：显示风险分数、检测方法、匹配类别
- ✅ **放行**：显示风险分数、检测方法

### 5. 📜 日志查看

显示最近的检测日志：
- 🔴 红色：被拦截的消息
- 🟢 绿色：放行的消息
- 包含：时间、用户、类别、分数等信息

## 快速配置示例

### 场景 1：只使用模式匹配（推荐）
```
1. 全局配置 → ML 权重：0
2. ML 模型 → 启用 ML 检测：取消勾选
3. Ollama → 启用 Ollama：取消勾选
4. 保存配置
```

### 场景 2：模式匹配 + ML 检测（英文环境）
```
1. 全局配置 → 拦截阈值：0.3
2. ML 模型 → 启用 ML 检测：勾选
3. ML 模型 → 模型选择：ProtectAI/deberta-v3-base-prompt-injection-v2
4. ML 模型 → 检测阈值：0.7
5. 保存配置
```

### 场景 3：三层全开（高安全环境）
```
1. 全局配置 → 拦截阈值：0.25
2. ML 模型 → 启用：勾选，阈值 0.65
3. Ollama → 启用：勾选，阈值 0.8
4. 保存配置
```

## 配置持久化

所有配置保存在：
```
~/.openclaw/workspace/Nash/src/config/config.json
```

修改会立即生效并持久化到文件。

## 故障排除

### Web UI 无法访问
```bash
# 检查服务状态
curl http://127.0.0.1:8768/health

# 重启服务
cd ~/.openclaw/workspace/Nash
pkill -f nash_server.js
./start.sh
```

### 配置无法保存
1. 检查配置文件权限
2. 查看服务日志：`tail -20 ~/.openclaw/workspace/Nash/logs/nash.log`

### ML 检测不工作
1. 检查 HuggingFace Token 是否设置
2. 检查网络连接
3. 查看日志中的 ML 加载信息

## 最佳实践

1. **从宽松开始**：初始阈值设高（0.4），逐步降低
2. **监控日志**：定期检查误报情况
3. **中文环境**：主要依赖模式匹配，ML 作为补充
4. **英文环境**：可以启用 ML 检测增强效果
5. **定期更新**：根据新出现的攻击模式更新关键词
